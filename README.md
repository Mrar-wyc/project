## 项目架构

### 整体设计

项目是一个 **纯前端 SPA（单页应用）**，无后端依赖。所有游戏逻辑（战斗引擎、经济、羁绊计算）均在前端完成。

- **数据层**（`src/data/`）—— 声明式游戏配置，英雄 / 羁绊 / 装备 / 投资策略均为纯数据
- **引擎层**（`src/engine/`）—— 自定义 ECS 战斗引擎，执行 AV 制回合战斗模拟
- **状态层**（`src/store/`）—— 唯一的 Zustand store 管理全部游戏状态
- **视图层**（`src/components/`）—— React 组件，通过 store actions 驱动，无直接状态变更
- **工具层**（`src/utils/`）—— 羁绊计算等纯函数工具

### 组件树

```
main.tsx
└── App.tsx
    ├── TopBar                  # 生命值 / 回合进度 / 金币 / 等级+EXP / 连胜
    ├── 主布局
    │   ├── 左侧区域
    │   │   ├── BattleBoard      # 敌方阵容 + 前台 4 槽 + 后台 6 槽
    │   │   ├── BenchPanel       # 备战席 9 槽
    │   │   └── InventoryPanel   # 装备背包（拖拽装备 / 合成）
    │   └── 右侧面板
    │       ├── InfoPanel        # 羁绊状态 / 策略投资列表 / 战斗按钮
    │       └── ShopPanel        # 招募商店（5 张卡牌 + 刷新 + 购买经验）
    └── EventModal               # 投资策略选择弹窗（每 5 节点触发）
```

### 数据流

```
用户操作 → Store Actions → 状态更新 → React 重渲染组件
                                ↓
                          战斗开始时
                                ↓
              CombatEngine.init(boardState)
                                ↓
              模拟战斗 → 回调更新 combatState
                                ↓
              战斗结束 → endCombat(win/lose)
```

## 核心玩法

### 游戏流程

游戏按照 **Planning → Combat → Event → Planning** 的循环推进：

```
Node 1    Node 2    Node 3    Node 4    Node 5
  🛒        ⚔️         🛒        ⚔️         🎲
Planning  Combat   Planning  Combat     Event
```

| 阶段 | 标识 | 玩家操作 |
|------|------|----------|
| `planning` | 准备阶段 | 购买英雄、调整阵容站位、合成装备、购买经验 |
| `combat` | 战斗阶段 | 全自动回合制战斗，观看 AV 时间线推进 |
| `event` | 事件阶段 | 从 3 个随机投资策略中选择 1 个（每 5 节点触发） |
| `gameover` | 游戏结束 | 生命值归零，展示结算界面 |

### 英雄系统

#### 英雄属性

每位英雄拥有 5 项基础属性：

| 属性 | 说明 |
|------|------|
| HP | 生命值，归零即阵亡 |
| Attack | 攻击力，影响普攻和技能伤害 |
| Defense | 防御力，减少受到的伤害 |
| Speed | 速度，决定 AV 行动顺序（Speed 越高行动越快） |
| Cost | 费用等级（1-5），决定购买价格和属性强度 |

#### 费用等级与概率

英雄按费用分为 5 个等级，高费 = 高属性。商店中不同费用英雄的出现概率随玩家等级动态变化，高级别更易刷出高费英雄。

| 费用 | 示例英雄 | 特性 |
|------|----------|------|
| 1 费 | 三月七、丹恒、青雀、佩拉 | 基础属性，容易三星 |
| 2 费 | 停云、素裳、娜塔莎、艾丝妲 | 过渡核心 |
| 3 费 | 姬子、瓦尔特、布洛妮娅、彦卿 | 中期主力 |
| 4 费 | 景元、希儿、卡芙卡、托帕 | 后期核心输出 |
| 5 费 | 银狼、饮月、刃、黑天鹅、阿格莱雅 | 终极 Carry |

#### 星级合成

- 规则：**3 张相同英雄 + 相同星级 → 1 张高一星级** 英雄
- 属性倍率：每升 1 星，属性 × 1.8
- 星级路线：**1★ → 2★ → 3★**（最高三星）

```
1★ 三月七 + 1★ 三月七 + 1★ 三月七 → 2★ 三月七（属性 ×1.8）
2★ 三月七 + 2★ 三月七 + 2★ 三月七 → 3★ 三月七（属性 ×3.24）
```

#### 共享卡池机制

- 所有玩家共享同一个英雄卡池
- 每种英雄卡池总量有限（`poolCounts` 追踪剩余数量）
- 卖出英雄会将卡牌返还卡池
- 顾问英雄（银狼 LV.999、加拉赫）**不进入初始卡池**，需通过投资策略解锁

### 羁绊系统

场上英雄根据携带的 **标签（traits）** 激活对应的羁绊效果。每位英雄拥有 1-2 个标签，当场上拥有相同标签的英雄数量达到激活阈值时，触发羁绊。

#### 羁绊计数规则

- 只统计 **场上**（棋盘前后台）的英雄
- 同一英雄 ID 的多张卡牌 **只计数一次**（鼓励上阵不同英雄）
- 支持多级羁绊：达到更高人数阈值，效果叠加

### 战斗引擎

战斗系统是一个 **自定义 ECS（Entity Component System）引擎**，基于 **AV（Action Value）制** 驱动回合战斗。

#### ECS 架构

```
ECSWorld
├── StatusComponent    # 实体属性（HP / ATK / DEF / SPD）
├── ActionComponent    # 行动值（currentAV）
├── FactionComponent   # 阵营（player / enemy）& 站位（frontline / backline）
├── BondComponent      # 羁绊标签列表
└── EnergyComponent    # 能量系统（current / max）
```

#### AV 制战斗流程

1. **初始化**：战斗开始时，所有实体 AV = 10000 / Speed（速度越快 AV 越小，行动越早）
2. **Tick 循环**：
   - 选取 AV 最小的实体 → 该实体执行行动
   - 消耗其 AV，推进全局时间线
   - 时间线达到 `maxAV（180）` 则超时判负
3. **行动类型**：
   - **前台单位**：直接攻击敌方
   - **后台单位**：提供光环效果 / 追击支援
4. **特殊机制**：
   - **神君**（仙舟 4 人）：召唤神君进行弹射攻击
   - **欢愉**（阿哈时刻）：后台积攒笑料点数，前台爆发
   - **持续伤害**：行动时对敌方造成基于最大生命值的真实伤害
   - **能量溢出**：能量满溢时重置 AV，实现连环行动
5. **胜负判定**：一方全部阵亡 → 判负 / AV 时间线超限 → 防守方判负

```
Speed 200 → AV = 50   → 最先行动（高速单位）
Speed 100 → AV = 100  → 第二行动
Speed 50  → AV = 200  → 最后行动（低速单位）
```

### 经济系统

#### 收入来源

| 来源 | 数值 |
|------|------|
| 每回合基础收入 | +5 金币 |
| 战斗胜利奖励 | +1 金币 |
| 利息收入 | 每 10 金币 → +1 金币（基础上限 3） |
| 连胜奖励 | 连续胜利可获得额外金币 |
| 出售英雄 | Cost × 3^(star-1) 金币 |

#### 支出项目

| 项目 | 费用 |
|------|------|
| 商店购买英雄 | 等于英雄 Cost 的金币 |
| 购买经验 | 4 金币 → 4 EXP |
| 商店刷新 | 2 金币 |
| 利息管理 | 保持 30+ 金币可吃满基础利息 |

#### 利息计算公式

```
利息 = min(floor(当前金币 / 10), 利息上限)
基础利息上限 = 3
选择「降本增效」后 → 利息上限提升至 9
```

### 投资策略

每 **5 个节点**（约 3 场战斗后）触发事件阶段，系统从投资策略池中 **随机抽选 3 个** 供玩家选择 1 个，效果持续整局游戏。

| 策略 | 等级 | 效果描述 |
|------|------|----------|
| 💰 **信用点扩张** | T1 | 立即获得 20 金币，但下一次商店刷新费用翻倍 |
| 📈 **高维数据降维** | T3 | 队伍等级上限 +1，但全体角色最大生命值降低 10% |
| ⚔️ **激进型风险对冲** | T2 | 连胜金币奖励 +1，但战斗失败扣除双倍生命值 |
| 🔄 **资产重组** | T2 | 立即免费刷新商店，且当前所有卡牌变为 4 费及以上 |

> T1 = 普通 / T2 = 稀有 / T3 = 史诗

## 数据驱动

游戏内容完全由数据配置驱动，无需修改组件代码即可扩展：

```
src/data/
├── heroes.ts        # 英雄配置文件（23 位角色）
├── bonds.ts         # 羁绊 / 命途配置（15 种羁绊，多级激活）
├── equipment.ts     # 装备配置（4 基础 + 4 高级，含合成配方）
└── investments.ts   # 投资策略配置（4 种策略，分 T1-T3）
```

### 配置接口

```typescript
// 英雄配置
interface HeroConfig {
  id: string;          // 'h1' ~ 'h23'
  name: string;        // 中文名称
  cost: number;        // 1-5 费用等级
  traits: string[];    // 羁绊标签（1-2 个）
  isAdvisor?: boolean; // 是否顾问英雄（不进入初始卡池）
  baseStats: {
    hp: number;
    attack: number;
    defense: number;
    speed: number;     // 影响 AV 计算
  };
}

// 羁绊配置
interface BondConfig {
  id: string;          // 英文 ID
  name: string;        // 中文名称
  levels: BondLevel[]; // 多级激活
}

interface BondLevel {
  count: number;       // 激活所需人数
  description: string; // 效果描述
  statsMultiplier?: {  // 属性倍率
    attack?: number;
    hp?: number;
    speed?: number;
    defense?: number;
  };
}

// 装备配置
interface EquipmentConfig {
  id: string;
  name: string;
  type: 'basic' | 'advanced';
  icon: string;
  description: string;
  stats?: Record<string, number>;
  recipe?: [string, string]; // 高级装备合成配方
}

// 投资策略配置
interface InvestmentConfig {
  id: string;          // 'inv_1' ~ 'inv_4'
  name: string;
  description: string;
  effect: (gameState: any) => void;
  tier: 1 | 2 | 3;    // 稀有度
}
```

## 视觉设计

整体采用 **HSR（Honkai: Star Rail）风格** 的深空暗黑科幻视觉语言。

### 配色方案

| 色彩 | 色值 | 用途 |
|------|------|------|
| 深空背景 | `#0A0E14` | 全局背景 |
| 金色镶边 | `#D4A853` | 边框、标题、强调元素 |
| 青蓝科技光 | `#4EE2EC` | 科技面板、能量效果、高亮 |
| 紫电 | `#C77DFF` | 伤害数字、特殊效果、稀有标识 |
| 暗金面板 | 半透明 + 模糊 | 卡片/面板背景（毛玻璃效果） |

### 视觉特效

- **粒子系统**：全局漂浮粒子动画，营造科技感
- **扫描线**：横向扫描光线掠过面板
- **呼吸光效**：关键元素脉冲呼吸动画
- **3D 卡牌**：hover 时卡牌倾斜（perspective + rotate 3D 变换）
- **几何裁剪**：菱形标签、六边形徽章、斜切状态栏
- **数值脉冲**：伤害数字和金币变动时的脉冲动画
- **战斗反馈**：浮动伤害数字、血条平滑动画、选中高亮环

### CSS 架构

```
Tailwind CSS 3 框架
  + CSS 自定义属性（主题色变量）
  + Framer Motion 动画（进入 / 退出过渡）
  + transform 3D 卡牌效果
```

## 开发指南

### 添加新英雄

1. 打开 `src/data/heroes.ts`
2. 在 `HEROES` 数组中按费用分组添加新配置：
   ```typescript
   { id: 'h24', name: '新角色', cost: 3, traits: ['仙舟', '巡猎'], 
     baseStats: { hp: 600, attack: 70, defense: 40, speed: 105 } }
   ```
3. 英雄 ID 遵循 `h{N}` 递增格式
4. 顾问英雄（需事件解锁）设置 `isAdvisor: true`
5. 新英雄将自动出现在商店卡池（非顾问英雄）和 InfoPanel 中

### 添加新羁绊

1. 打开 `src/data/bonds.ts`
2. 在 `BONDS` 对象中添加配置：
   ```typescript
   '新羁绊名': {
     id: 'new_bond',
     name: '新羁绊名',
     levels: [
       { count: 2, description: '效果描述', statsMultiplier: { ... } }
     ]
   }
   ```
3. 打开 `src/engine/CombatEngine.ts`，在 `init()` 方法中实现羁绊的战斗逻辑
4. InfoPanel 会自动显示新羁绊，无需修改视图代码

### 添加新装备

1. 打开 `src/data/equipment.ts`
2. 基础装备：添加配置，`type: 'basic'`
3. 高级装备：添加配置，`type: 'advanced'`，并指定合成配方：
   ```typescript
   recipe: ['basic_sword', 'basic_crystal'] // 配方：铁剑 + 水晶
   ```
4. InventoryPanel 中的拖拽合成逻辑自动处理新配方

### 添加新投资策略

1. 打开 `src/data/investments.ts`
2. 在 `INVESTMENTS` 数组中添加策略配置
3. 在 `effect` 函数中实现策略效果（修改 gameState）
4. 策略将自动进入事件阶段的选择池

### 状态管理规范

| 规则 | 说明 |
|------|------|
| 统一入口 | 所有状态修改通过 `useGameStore` 暴露的 actions 进行 |
| 禁止直接修改 | 不可直接修改 store 中的嵌套对象，必须用对应的 action |
| UUID 生成 | `HeroInstance.uuid` 使用 `Math.random().toString(36).substring(7)` |
| 阶段状态 | 游戏阶段使用 `'planning' | 'combat' | 'event' | 'gameover'` 精确字符串 |
| 不可变更新 | Store actions 内部使用 Zustand 的不可变更新模式 |

### TypeScript 规范

- 启用严格模式（`tsc -b` 检查所有类型）
- 所有组件和函数需显式类型注解
- 数据配置文件导出类型接口供外部使用
